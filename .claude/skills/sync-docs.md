# 文档同步规则

修改或新增项目中的任何代码时，**必须**同步更新以下两个文件：

## 1. md/rz.md — 开发日志

记录**做了什么、为什么这么做、遇到的问题及解决方案**：

- **新增功能**: 记录完成内容、涉及的文件、关键实现细节
- **Bug 修复**: 记录问题现象、根因、修复方案
- **架构决策**: 记录选择某方案的原因、权衡考虑
- **后续注意事项**: 记录容易踩坑的点、约定、约束

格式参照已有日志条目：
```markdown
## YYYY-MM-DD — 标题

### 完成内容
- 具体改动项

### 架构决策 / 遇到的 Bug 及解决方案
- 决策或问题描述 + 原因/修复

### 后续注意事项
- 需要注意的点
```

## 2. md/CLAUDE.md — 项目文档

记录**项目结构、系统设计、架构信息**的变更：

- **文件结构**: 新增/删除文件时更新目录树和依赖关系说明
- **核心系统设计**: 新增系统或修改现有系统行为时更新对应章节
- **游戏状态机**: 状态变更时更新状态转换图
- **API/接口**: 新增或修改公开接口时更新文档

格式参照现有文档结构，保持章节层级一致。

## 触发条件

以下操作**必须**触发文档同步：
- 新增 `.py` 文件
- 修改现有 `.py` 文件中的类、函数、常量
- 新增或修改关卡数据
- 新增或修改 UI 界面
- 修复 Bug

## 执行顺序

1. 完成代码修改
2. 更新 `md/rz.md`（开发日志）
3. 更新 `md/CLAUDE.md`（如有架构/结构变更）
4. 将文档变更一并提交
